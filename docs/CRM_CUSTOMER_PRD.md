## 一、概述

### 1.1 产品定位
面向货代企业内部的销售管理系统，实现客户资源集中管理、销售流程规范化、**防止重复客户新增**。

### 1.2 用户角色
| 角色 | 职责 |
|-----|------|
| **管理员** | 客商分配、数据管理、强制新增（跳过重复校验） |
| **销售主管** | 团队客户管理、业绩监控、客户转移 |
| **销售人员** | 客户开发、跟进维护、公海认领 |

### 1.3 系统边界
- **后台系统**（本文档）：完整的客商管理功能
- **前台会员系统**（Phase 2）：会员注册时关联客商数据，不可重复新增
- **关联系统**：读取订舱/运价数据，不替代订单系统

### 3.1 客商管理

#### 3.1.1 客商列表

| 功能 | 说明 |
|-----|------|
| 搜索 | 按公司名称、统一信用代码、联系人、电话搜索 |
| 高级筛选 | 客户等级、客户类型、归属销售、跟进状态、创建时间 |
| 列表字段 | 公司名称、统一信用代码、联系人、客户等级、归属销售、最后跟进时间、状态 |

| 快捷操作 | 查看详情、新增跟进、编辑、删除（管理员） |

**状态说明：**
- **正常**：已分配给销售，正常跟进中
- **公海**：待分配/已回收，可被认领
- **归档**：停止合作，保留记录

#### 3.1.2 新增客商（⚠️ 重复校验重点）

**入口**：客商列表 → [新增客商]

**新增流程：**
```
填写客商信息
    ↓
系统校验重复（实时）
    ↓
├─ 发现重复 ──→ 提示重复信息，阻止新增
│               显示：重复客商名称、归属销售、联系方式
│               操作：查看详情 / 联系归属销售 / 取消
    ↓
└─ 无重复 ────→ 选择归属销售 / 放入公海
                ↓
              保存成功
```

**重复校验规则：**

| 校验字段 | 校验方式 | 重复判定 |
|---------|---------|---------|
| 统一社会信用代码 | 精确匹配 | 完全一致即为重复 |
| 公司名称 | 模糊匹配 | 相似度 > 85% 或包含关系 |
| 联系人电话 | 精确匹配 | 主联系人电话重复 |

**重复提示界面：**
```

┌────────────────────────────────────────────────────────┐

│    统一代码：91330212MA2J7...                          │
│    归属销售：张三                                       │
│    最后跟进：2025-03-20                                │
│    [查看详情] [联系归属销售]                            │
├────────────────────────────────────────────────────────┤
│ 2. 宁波海运集团                                         │
│    相似度：72%                                          │
│    归属销售：李四                                       │
```

### 3.2 公海池

**功能说明：**
存放未被认领或已回收的客户资源，供销售认领或管理员分配。

**进入公海的情况：**
1. 新增客商时选择"暂不分配"
2. 管理员手动移入公海
3. 销售离职/调动，客户未交接
4. **自动回收**：超过N天无跟进记录（可配置，默认30天）

**公海池列表：**
- 筛选：在库时长、客户类型、来源
- 字段：公司名称、入库时间、在库时长、原归属销售
- 操作：认领、分配（管理员）、查看详情

**认领规则：**
1. 销售点击[认领]
2. 系统再次校验重复（防止同时认领）
3. 认领成功 → 归属变为该销售，状态变为"正常"
4. 认领失败 → 提示已被他人认领

### 3.3 客户分配

**管理员功能：**
- 将公海客户分配给指定销售

**分配记录：**
- 记录分配历史
- 支持查看某客户的归属变更轨迹

**跟进表单：**

- 跟进时间（默认当前）
- 跟进方式：电话、微信、邮件、拜访、展会、其他
- 跟进内容（富文本，支持简单排版）
- 客户反馈
- 客户意向等级变化（选填）
- 下次跟进提醒时间（选填，到期首页提醒）
- 同步更新画像（勾选）

**跟进列表（我的跟进）：**
- 时间倒序展示
- 筛选：按客商、时间范围、跟进方式
- 快捷跳转客商详情

**首页提醒：**

### 3.5 统计分析

### 4.1 核心实体

**Customer（客商）**
```typescript
{
  id: string

  // 基础信息
  name: string                    // 公司名称 *
  creditCode: string | null       // 统一社会信用代码（唯一索引）
  industry: string | null         // 行业
  scale: string | null            // 公司规模
  registeredAddress: string | null // 注册地址

  // 业务信息

  preferredRoutes: string[]       // 偏好航线
  settlementPreference: string    // 结算方式偏好

  requirements: string            // 特殊需求
  competition: string             // 竞争情况
  tags: string[]                  // 标签

  // 管理
  status: 'active' | 'public' | 'archived'  // 状态
  ownerId: string | null          // 归属销售ID
  ownerHistory: OwnerHistory[]    // 归属变更历史
  source: string                  // 来源：manual/import/convert

  // 联系人（嵌入式或关联）
  primaryContactId: string | null // 主联系人ID

  // 时间戳
  createdAt: Date
  updatedAt: Date
  lastFollowUpAt: Date | null     // 最后跟进时间
  nextFollowUpAt: Date | null     // 下次跟进提醒
  autoPublicAt: Date | null       // 自动回收时间（预留）

interface OwnerHistory {
  userId: string
  userName: string
  action: 'assign' | 'transfer' | 'claim' | 'release'

  createdAt: Date
```

### 4.2 唯一性约束

```sql
-- 统一社会信用代码唯一（允许NULL，但不允许重复非NULL）
CREATE UNIQUE INDEX idx_customer_credit_code ON Customer(creditCode) WHERE creditCode IS NOT NULL;

-- 公司名称索引（用于搜索和相似度校验）
CREATE INDEX idx_customer_name ON Customer(name);

-- 归属销售索引
CREATE INDEX idx_customer_owner ON Customer(ownerId);
```

### 4.3 重复检测算法（伪代码）

```typescript
async function checkDuplicateCustomer(input: {
  name: string
  creditCode?: string
  phone?: string
}): Promise<DuplicateResult[]> {
  const results: DuplicateResult[] = []

  // 1. 统一信用代码精确匹配（权重最高）
  if (input.creditCode) {
    const byCreditCode = await db.customer.findUnique({
      where: { creditCode: input.creditCode }
    })
    if (byCreditCode) {
      results.push({ customer: byCreditCode, matchType: 'credit_code', score: 100 })
    }
  }

  // 2. 公司名称相似度匹配
  const byName = await db.customer.findMany({
    where: {
      name: { contains: input.name.slice(0, 4) } // 先模糊筛选减少计算
    }
  })

  for (const c of byName) {
    const score = calculateSimilarity(input.name, c.name)
    if (score > 85) {
      results.push({ customer: c, matchType: 'name', score })
    }
  }

  // 3. 主联系人电话匹配
  if (input.phone) {
    const byPhone = await db.contact.findFirst({
      where: { phone: input.phone, isPrimary: true },
      include: { customer: true }
    })
    if (byPhone) {
      results.push({ customer: byPhone.customer, matchType: 'phone', score: 90 })
    }
  }

  return results
}
```

### 5.2 数据隔离规则

1. **销售只能看到：**
   - 归属为自己的客商（`ownerId = 自己`）
   - 公海池客商（`status = 'public'`）
   - 不能看到其他销售的客商详情（除非通过防撞客查询）

2. **销售主管可以看到：**
   - 自己团队下所有销售的客商
   - 公海池客商

3. **管理员可以看到：**
   - 全部客商数据

## 七、系统集成

### 7.1 与订单系统关联

**成交记录读取：**
```sql
-- 通过 customerId 关联查询
SELECT
  b.id as bookingNo,
  b.route,
  b.containerType,
  b.containerCount,
  b.amount,
  b.status,
  b.createdAt
FROM Booking b
WHERE b.customerId = :customerId
ORDER BY b.createdAt DESC
```

### 7.2 与会员系统关联（Phase 2）

**数据流向：**
```
后台客商管理                    前台会员中心
     │                              │
     ├─ 已有客商 ─────────────────→ 会员注册时关联选择
     │                              │
     ←─ 新注册会员 ───────────────┤
          （审批后转为正式客商）      │
```

**关联字段：**
- 会员注册时需要选择/填写所属公司
- 系统根据公司名称匹配已有客商
- 匹配成功 → 关联到该客商
- 匹配失败 → 创建新客商（放入公海待审核）

### 7.3 与现有用户系统集成

- 复用现有用户登录/权限系统
- 销售用户即为系统中的 `User`（role = 'sales'）
- JWT Token 中包含用户角色信息

## 十、附录

### 10.1 客户等级标准

| 等级 | 名称 | 标准 | 维护频率 |
|-----|------|------|---------|
| A | 核心客户 | 月货量 > 100TEU 或 月营收 > 50万 | 每周 |
| B | 重要客户 | 月货量 30-100TEU 或 月营收 15-50万 | 每两周 |
| C | 一般客户 | 月货量 5-30TEU 或 月营收 3-15万 | 每月 |
| D | 潜在客户 | 有合作意向，尚未成交或成交<5TEU/月 | 按需 |

### 10.2 客户标签建议

- **价值类**：重点客户、长期合作、高潜力、战略客户
- **类型类**：直客、同行、海外代理、国内代理
- **状态类**：新客户、谈判中、已合作、流失风险、已流失
- **业务类**：FCL偏好、LCL偏好、空运偏好、门到门需求
- **结算类**：现结、月结、季结、账期客户
